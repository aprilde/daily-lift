  up.

## Step B - Data models + persistence (complete)

Built the data layer: serializable models, seed workout content, and two
persistence stores (workout plan as a JSON file, today's completion state in
`SharedPreferences`) - no UI changes yet.

**Dependencies:** added the `kotlin.plugin.serialization` plugin (alias in
`gradle/libs.versions.toml`, applied in `app/build.gradle.kts`) plus
`kotlinx-serialization-json` as an `implementation` dependency.

**Data models (`app/src/main/java/com/dailylift/app/data/`):** `Exercise`
(`id`, `name`, `weight`, `reps`, `tip`, `imageStartRef`, `imageEndRef`),
`WorkoutDay` (`focus` + `List<Exercise>`), `Weekday` enum (Monday-Sunday), and
`WorkoutData` as a `Map<Weekday, WorkoutDay>` typealias. `Completion` holds
today's `date` plus a `done: Map<String, Boolean>` keyed by `Exercise.id`
(not list position), resetting whenever `date` no longer matches today. All
marked `@Serializable`.

**Seed data (`SeedWorkoutData.kt`):** `createSeedWorkoutData()` ports the
Monday-Friday plan from `workout-widget-prototype.html`'s `DEFAULT_DATA`
(Decisions 2-4): "AMRAP" replaced with "as many as you can" (B2), every
exercise gets a generated UUID `id` (B3), and `imageStartRef`/`imageEndRef`
are derived from a slugified exercise name (B5). Saturday/Sunday have no
entries (B4).

**`WorkoutDataStore`:** reads/writes `WorkoutData` as pretty-printed JSON in
`workout_data.json` inside a `File` directory. Takes a plain `File` rather
than a `Context` so it's unit-testable without an Android runtime (and reusable
from a future widget process). `loadOrSeed()` returns existing data or seeds,
persists, and returns the default plan.

**`CompletionStore`:** reads/writes today's `Completion` as a single JSON
string under one `SharedPreferences` key.

**Unit tests (12 total, all passing):**
- `ModelSerializationTest` - B1: JSON round-trip for `Exercise`, `WorkoutDay`,
  `WorkoutData`, and `Completion` (4 tests).
- `SeedWorkoutDataTest` - B2 (no literal "AMRAP"), B3 (unique non-empty ids),
  B4 (Monday-Friday only, no weekend entries), B5 (non-empty image refs) (4
  tests).
- `WorkoutDataStoreTest` - B6: save then reload from a fresh store over the
  same directory ("app restart") matches the saved data, plus two supporting
  cases for `load()` returning `null` before anything is saved and
  `loadOrSeed()` persisting its seed for the next load (3 tests).
- Plus the existing `ExampleUnitTest` (1 test).
